## Verified Facts

Architecture (→ CLAUDE.md § Architecture):

- Layered crates; dependencies flow downward only.
- Gameplay domain traits are async, with blanket impls on `HasContext`; the top-level trait composes all domains.
- Internal helpers are suffixed `_impl` and take `C: ConnectionTrait`, so they can join transactions started by public trait methods and be reused across modules.
- `Codex`: read-only in-memory manifest snapshot, loaded at startup, never mutates — single source of truth for game config.
- DB scope split: `entity::user` (accounts / tokens) vs `entity::profile` (per-player state).
- KCSAPI responses use the `svdata=` prefix; handlers return `KcApiResponse`.

Code style (→ CLAUDE.md § Code Style):

- Soft tabs, 4 spaces. `.rustfmt.toml` is `hard_tabs = false`, `tab_spaces = 4`; `.editorconfig` agrees. (The old `openspec/config.yaml` "Hard tabs" line was a typo and is removed with openspec.)
- `unsafe_code` denied workspace-wide; `missing_docs` warned.
- Balance / numeric `Default` changes follow the Balance Defaults Policy: own commit, `feat(balance):` / `chore(balance):`, previous values in the body, a `docs/plans/` plan (ce-plan), regression test.

Do-not-modify (→ CLAUDE.md § Do-Not-Modify Files):

- `crates/emukc_bootstrap/assets/*.json` and `main-decoder/out/battle/*.json` are generated by `bun run decode` → `--sync-battle-assets`; never hand-edit.
- `tests/gameplay_tests/battle_golden.rs` is a frozen deterministic transcript.

- **OPEN (plan 003 scope, boss-node-equivalence design change)** — 2 stable-failing boss-path tests in `tests/sortie_battle.rs`: `sortie_battle_result_advances_boss_quest_on_real_boss_node` (line 481) and `repo_wikiwiki_asset_supports_real_map_boss_progression` (line 514). **Corrected diagnosis 2026-06-22 (earlier "codex topology error" claim was WRONG):** the kcdata route→cell model is INTENTIONAL and matches the real KanColle API (real capture `map_1-2.json` has `api_no: 0–6` = 7 cells = 7 routes; tests `convergent_routes_preserve_duplicate_route_cells` etc. deliberately protect it). The real defect is narrow: a boss NODE targeted by multiple routes produces multiple boss cells (map 1-2 node E = cell 5 via A→E AND cell 6 via D→E), but `boss_cell_no: i64` captures only the first (5). So reaching cell 6 (also node E, also boss) is not recognized as "boss reached": `path_to_boss` DFS targets only cell 5 → cell 4 (routes to cell 6) reports unreachable; the test's `while current != boss` never matches cell 6. Confirmed via DEBUG_BOSS instrumentation (now removed): `next_sortie(5)` from cell 1 errors "not a valid route" (routing rules only permit cell 4), the path 1→4→6 lands on cell 6 which is node E but ≠ boss_cell_no. **Correct fix = boss-node equivalence**: treat all cells whose `node_label` equals the boss node's label as boss, not just the first route's cell. This is a design change touching `boss_cell_no` semantics across sortie end-detection, quest advancement, `path_to_boss`, and the battle validator — needs scoped effort, not a quick patch. A parser rewrite that collapses same-node cells was tried and REVERTED (broke 5 kcdata tests protecting real-API numbering). This is plan 003 U1.2/U3.1 scope.
